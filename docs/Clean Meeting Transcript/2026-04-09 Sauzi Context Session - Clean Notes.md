**Duration:** ~6 minutes

## Summary

Nick outlined the next phase of the Sauzi.ai website — moving from the initial "lift and shift" of the Fractional site to a fully branded experience built around a **chef/culinary theme**. He also identified the agent and skill infrastructure needed to support ongoing website development and content marketing.

---

## Key Points

### 1. Website Status & Brand Direction

- Sauzi.ai is live. The current site was a lift-and-shift of the prior Fractional website with updated branding.
- The next phase expands on the brand concept: **Sauzi as a chef orchestrating a meal** — where the meal is the client's data infrastructure.
- The website experience should feel like reading a **menu**: visitors see what's on offer, how ingredients (tools/platforms) are combined, and what the finished dish (AI data infrastructure) looks like.
- Sauzi's positioning: **data team as a service**, building AI data analysis infrastructure for clients.

### 2. Agents Needed for Website Development

Four agents are needed to support the website build:

| Agent | Purpose |
|---|---|
| **UI/UX Agent** | Optimize and design website pages consistent with the chef/brand theme |
| **Content Agent** | Write brand-aligned copy, blog posts, and page content |
| **SVG Agent** | Generate SVG images — banners, thumbnails, blog post graphics |
| **Brand Orchestrator Agent** | Ensure all elements maintain a fully cohesive, branded experience |

A shared set of **image standards skills** is also needed to enforce consistent dimensions, pixel counts, and brand color usage across generated images.

### 3. Marketing Agents

A separate layer of agents to handle content marketing:

- Write blog posts tied to the brand narrative
- Generate LinkedIn posts linked to blog content
- Incorporate blog content into **playbooks** and outbound messages

### 4. Playbooks Feature

- High priority: a dedicated **Playbooks page** that shows how Sauzi builds AI agents for specific client tech stacks (e.g., Shopify + Recharge + BigQuery).
- Each playbook should display the full data flow in a clear, visual UI.
- Should be accessible to both technical and non-technical visitors.
- Example flow: "Do you have a data warehouse? Do you have an ETL platform? No — we can integrate Fivetran for you."
